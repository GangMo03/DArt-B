# 1️⃣ 개념정리

## 01. 자꾸만 늘어가는 '방법맨'이란

```
✅ 학습 목표 :
* '방법맨'이란 무엇인지 설명할 수 있다.
* 문제 해결보다 기법 자체에 집착하게 되는 사고 방식의 위험성을 설명할 수 있다. 
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->

가장 인상 깊었던 문장이 "곧바로 '해결 방안'에 무턱대고 달려드는 것'이 가장 큰 문제다" 였다. 확실히 지금까지 분석을 본격적으로 공부하기 전에는 뭔가 기술적 방법에 대해 더욱 집착했던 것 같다. 하지만 여기서 '원인'이 가장 중요하다고 말하듯이 왜 이런 문제가 발생하는지 구조적으로 잘 파악하는 것이 '방법맨'을 피하기 위해 가장 중요한 것 같다.

## 02. 방법맨이 돼버리는 구조

```
✅ 학습 목표 :
* 방법맨이 되어버리는 구조적 요인을 이해한다.
* 분석 기법보다 '문제 인식 -> 사고 구조 설계'가 우선임을 인식한다. 
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->

'무엇이 결정적 원인인가'와 '무억을 해결, 실현하고자 하는가'를 확실히 한 다음에 해결방안인 '무엇을 해야 하는가'를 고민하여야 한다. 이 아이디어도 앞서 생각 했던 원인과 목적 및 문제와 논리적으로 맞는 방안을 제시해야지 갑자기 '참신한 아이디어'를 찾아 그동안 쌓아 왔던 논리 구조를 잃어버리면 안된다.



## 03. '왜(원인)' 그런지 끈질기게 생각하라

```
✅ 학습 목표 :
* 기법을 적용하기 전에 '무엇을, 왜 분석하는가'를 지속적으로 질문하는 태도를 기른다.
* 단순 결과에 안주하지 않고 원인을 집요하게 파고드는 분석 사고력을 기른다. 
```

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->

원인을 생각할 떄 겉으로 보이는 표면적인 원인에서 분석이 끝나면 안된다. 왜?라고 끊임없이 생각하며 깊이 파고들어야 한다.
